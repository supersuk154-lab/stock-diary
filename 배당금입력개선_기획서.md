# 배당금 기록 UX 개선 기획서

## 배경 및 목표

현재 `diary_inventory.py`의 배당금 직접 기록 폼은 종목명을 **자유 텍스트**로 입력받는다.
이미 보물함(`get_real_inventory`)에 종목 목록이 있는데 활용되지 않고, 오탈자·불일치 문제가 생길 수 있다.

**목표**: 보유 종목 드롭다운 → AI 종목명 확인 → 저장의 3단계로 UX를 개선한다.

---

## 개선 전 / 후 비교

| 항목 | 현재 | 개선 후 |
|------|------|---------|
| 종목 입력 | 자유 텍스트 | 보유 종목 드롭다운 (1순위) |
| 오탈자 방지 | 없음 | 드롭다운 선택 시 자동 방지 |
| 미보유 종목 | 동일 텍스트 박스 | 하단 별도 "직접 입력" 폼 |
| AI 검증 | 없음 | 직접 입력 시 AI가 종목명 정규화 |
| 저장 흐름 | 즉시 저장 | AI 확인 → 사용자 승인 → 저장 |

## 상세 구현 스펙

### A. 드롭다운 폼 (1순위)

**데이터 소스**: `get_real_inventory(user_id, supabase)` — 이미 존재하는 함수.  
반환값: `[{"종목": "삼성전자", "수량": 10, "평단가": 70000, "통화": "KRW"}, ...]`

**드롭다운 옵션 구성**:
```python
portfolio = get_real_inventory(user_id, supabase)
stock_options = [item["종목"] for item in portfolio]
# 통화는 보유 종목의 통화를 자동 선택 (선택 변경 시 currency도 자동 반영)
```

**자동 통화 선택**:
- 드롭다운에서 종목 선택 시 해당 종목의 `통화` 값을 `div_currency` 기본값으로 자동 설정
- 사용자가 변경 가능하도록 radio는 유지

**저장 로직**: 현재와 동일 (trades 테이블에 INSERT)

---

### B. 직접 입력 폼 + AI 검증 (2순위)

**단계별 세션 상태 관리**:

```python
# 세션 키
"_div_manual_input"       # 사용자가 입력한 원문 종목명
"_div_ai_result"          # AI가 정규화한 종목명 (확인 대기 중)
"_div_manual_confirmed"   # True: AI 확인 완료, 저장 폼 활성화
```

**단계 1 — 텍스트 입력 + AI 검증 버튼**:
```python
manual_input = st.text_input("종목명 (직접 입력)", placeholder="예) 삼전, S&P ETF, SCHD 등 편하게 쓰세요")
if st.button("🤖 AI 종목명 확인하기"):
    # AI 호출 → 정규화 결과 세션에 저장 → rerun
```

**AI 프롬프트 (간단하게)**:
```
사용자가 입력한 "{manual_input}"이 어떤 주식 종목인지 파악하여
정식 종목명(한국 주식은 "종목명(티커)", 해외 주식은 영문명)으로 답하시오.
확실하지 않으면 가장 가능성 높은 종목명 한 가지만 제시하고,
JSON 형식으로 {"name": "정식종목명", "reason": "한줄 근거"} 만 출력하시오.
```

**단계 2 — AI 확인 UI**:
```python
ai_result = st.session_state.get("_div_ai_result")
if ai_result:
    st.info(f'🤖 **"{ai_result["name"]}"** 을 말씀하신 게 맞나요?\n\n{ai_result["reason"]}')
    col1, col2 = st.columns(2)
    with col1:
        if st.button("✅ 맞아요, 이 이름으로 저장"):
            st.session_state["_div_manual_confirmed"] = True
            st.rerun()
    with col2:
        if st.button("✏️ 다시 입력"):
            # 세션 초기화
            st.session_state.pop("_div_ai_result", None)
            st.rerun()
```

**단계 3 — 확인된 종목명으로 금액 입력 + 저장**:
```python
if st.session_state.get("_div_manual_confirmed"):
    confirmed_name = st.session_state["_div_ai_result"]["name"]
    st.success(f"✅ 종목: {confirmed_name}")
    # 금액 입력 + 저장 폼 (드롭다운 폼과 동일 구조)
```

## 엣지 케이스 처리

| 상황 | 처리 방법 |
|------|---------|
| 보유 종목이 없음 (빈 포트폴리오) | 드롭다운 폼 숨기고 직접 입력 폼만 표시 |
| AI가 종목명 파악 불가 | `{"name": null, "reason": "..."}` → "파악하지 못했어요. 정식 종목명으로 다시 입력해주세요." |
| 드롭다운 선택 후 금액 0 입력 | 저장 버튼 비활성화 또는 경고 |
| 직접 입력에서 드롭다운과 동일 종목 입력 | 저장 후 정상 동작 (중복 허용, 배당은 여러 번 들어올 수 있음) |
| AI API 오류 | 오류 메시지 표시 + "AI 없이 그냥 저장" 버튼 fallback 제공 |

## 미결 사항 (구현 전 확인 필요)

1. **AI 클라이언트 전달 방법**  
   현재 `render_inventory_section(supabase, user_id, zen_mode)` 시그니처에 `ai_client`가 없음.  
   → `tab_diary.py`에서 `ai_client`를 추가 인자로 넘겨야 함.

2. **직접 입력 폼의 통화 기본값**  
   직접 입력 종목은 보유 종목이 아니므로 기본값 KRW? 또는 사용자가 선택?  
   → KRW 기본값, radio로 변경 가능하게 유지.

3. **AI fallback 저장 시 종목명 처리**  
   AI API 오류 시 "AI 없이 저장" 클릭하면 사용자가 입력한 원문 그대로 저장.

전반적으로 잘 구조화된 Streamlit 앱입니다. Supabase + Gemini API 연동, RLS 보안, 모바일 최적화 UI 등 많은 부분이 훌륭합니다. 하지만 아래와 같은 치명적 오류, 보안 취약점, UX 결함이 발견되어 수정이 필요합니다.

🔴 1. 데이터베이스 스키마 누락 (치명적 오류)
문제
supabase_schema.sql에 trades 테이블의 dividend_amount 컬럼이 없습니다.
그런데 diary_inventory.py에서 배당금 저장 시 이 컬럼을 사용합니다.

python
# diary_inventory.py (render_inventory_section → 배당금 직접 기록하기)
supabase.table("trades").insert({
    ...
    "dividend_amount": div_amount   # ← 컬럼 없음
}).execute()
➡️ 실행 시 오류 발생: column "dividend_amount" of relation "trades" does not exist

✅ 해결
supabase_schema.sql에 다음 ALTER 문을 추가하세요 (또는 Supabase SQL Editor에서 실행).

sql
-- trades 테이블에 dividend_amount 컬럼 추가
alter table trades add column if not exists dividend_amount float;
🔴 2. get_past_context 보안 취약점 (사용자 데이터 노출 가능성)
문제
db.py의 get_past_context 함수는 user_id가 빈 문자열이면 조건 없이 쿼리를 실행합니다.

python
def get_past_context(tags, supabase, user_id: str = ""):
    ...
    if user_id:
        query = query.eq("user_id", user_id)   # ❌ user_id가 ""이면 모든 사용자 데이터 조회
로그인 후에는 st.session_state["user_id"]가 항상 존재하지만,
예외 상황(예: 세션 만료 직후)에서 빈 문자열이 전달될 가능성이 있습니다.
RLS가 있더라도 명시적 조건을 생략하는 것은 위험한 습관입니다.

✅ 해결
user_id가 없으면 즉시 빈 문자열 반환하도록 수정하세요.

python
def get_past_context(tags, supabase, user_id: str = ""):
    if not user_id:
        return ""
    if not tags:
        return ""
    # ... 나머지 로직 (user_id 조건을 항상 추가)
    query = query.eq("user_id", user_id)   # 항상 실행
동일한 함수가 diary_chat.py와 diary_upload.py에서 호출되므로,
호출하는 쪽에서도 user_id가 없으면 호출하지 않는 것이 안전합니다.

🟡 6. 성능 최적화: 주가 캐시 TTL 부적절
문제
prices.py의 get_realtime_prices_bulk는 TTL=3600초(1시간)입니다.
장중에는 15분 단위로 time_bucket이 바뀌지만, 캐시는 1시간 동안 유지되어 15분마다 갱신되지 않음.

✅ 해결
장중 TTL을 900초(15분)로 줄이고, 장외는 3600초로 유지하도록 동적 TTL 적용.

python
def _get_ttl() -> int:
    now = datetime.datetime.now(KST)
    h, m = now.hour, now.minute
    if (9, 0) <= (h, m) < (15, 30):   # 한국 장중
        return 900   # 15분
    elif h >= 22 or h < 5:             # 미국 장중
        return 900
    else:
        return 3600  # 1시간

# 데코레이터에서 ttl 인자 대신 None으로 두고 함수 내부에서 캐시 키에 time_bucket 포함?
# 더 간단한 방법: @st.cache_data(ttl=900)으로 통일해도 무방.
